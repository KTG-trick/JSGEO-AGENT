# 支撑内容生成

## 目标

基于问题池和信源发现结果，生成更容易被 AI 理解和引用的支撑内容。

## 为什么需要支撑内容？

AI 在回答推荐类问题时，会引用网上的内容。如果我们提前在高权重平台发布优质内容，AI 就更可能引用这些内容来推荐我们的企业。

支撑内容不是泛泛营销文章，而是为 AI 推荐提供可引用事实和推荐理由。

## 输入数据

| 数据 | 来源 | 必需 | 说明 |
|------|------|------|------|
| 企业 profile | enterprise_profiles | 是 | 公司信息、产品服务 |
| RAG top chunks | knowledge_chunks | 是 | 知识库中的相关内容 |
| AI 问题池 | geo_question_sets | 是 | 用户可能问的问题 |
| 信源发现结果 | geo_source_discoveries | 是 | 高权重发布渠道 |
| 已确认规则 | evolution_rules | 否 | 之前优化周期的经验 |

## 内容类型

### 1. 咨询类文章

**目标**：回答用户怎么选、注意什么、怎么判断

**适用场景**：
- 用户在选择时有困惑
- 用户需要避坑指南
- 用户需要决策建议

**示例标题**：
- 《成都汽车音响改装怎么选？这份指南帮你避开 5 个坑》
- 《选择汽车隔音店要注意什么？资深车主的经验分享》
- 《新能源汽车改装方案怎么选？3 个关键因素》

**内容结构**：

```
1. 用户问题引入
   - 描述用户的困惑
   - 引出文章主题

2. 选择标准/注意事项
   - 列出 3-5 个关键标准
   - 每个标准有详细说明

3. 常见误区
   - 列出 2-3 个常见坑
   - 说明如何避免

4. 推荐方案（自然引出目标企业）
   - 介绍推荐方案
   - 自然引出目标企业优势

5. 总结
   - 总结关键点
   - 给出行动建议
```

### 2. 测评类文章

**目标**：对比方案、列出标准、自然引出目标企业优势

**适用场景**：
- 用户需要对比多个选项
- 用户需要客观评测
- 用户需要排名参考

**示例标题**：
- 《2024年成都汽车音响改装店测评：5家店横向对比》
- 《新能源汽车隔音方案测评：哪种最适合你的车型？》
- 《速冻食品品牌测评：口感、价格、品质全面对比》

**内容结构**：

```
1. 测评背景
   - 说明测评目的
   - 介绍测评对象

2. 评测标准
   - 列出评测维度
   - 说明评分依据

3. 对比方案
   - 逐个介绍方案
   - 列出优缺点

4. 结果分析
   - 综合对比
   - 得出结论

5. 推荐结论
   - 推荐方案
   - 说明理由
```

### 3. 推荐理由卡

**目标**：用结构化方式总结企业为何值得推荐

**适用场景**：
- 快速展示企业优势
- 提供结构化推荐理由
- 便于 AI 摘取

**示例格式**：

```
【推荐理由】
企业：成都行乐音改
推荐指数：⭐⭐⭐⭐⭐

核心优势：
1. 专注无损改装，不破坏原车线路
2. IASCA 认证调音师，专业度高
3. 新能源车型专属方案，技术领先

适合人群：
- 追求音质提升但不想破坏原车的车主
- 新能源车主
- 预算 1500-8000 元的用户

用户评价：
"改完音响效果提升明显，而且没有异响，很满意"
```

### 字段说明

| 字段 | 说明 |
|------|------|
| article_type | 内容类型：consulting（咨询类）、review（测评类）、recommendation_card（推荐理由卡） |
| title | 文章标题，贴近目标问题 |
| content | 完整的文章内容 |
| target_question | 这篇文章针对的问题 |
| publish_target | 建议发布的平台 |
| outline | 文章大纲 |
| facts_used | 使用的企业事实 |
| sources_to_reference | 建议引用的来源 |
| missing_facts | 缺失的事实（需要补充） |

## 质量标准

### 好的支撑内容

- 标题贴近问题池
- 内容引用企业知识库事实
- 结构清晰，适合 AI 摘取
- 每篇内容有引用依据
- 不编造案例和资质

### 坏的支撑内容

- 编造了案例和资质
- 标题和内容不相关
- 结构混乱，不适合 AI 摘取
- 没有引用依据
- 纯营销内容，没有实用价值

## 约束

1. 必须引用企业知识库事实
2. 不得编造案例和资质
3. 标题贴近问题池
4. 内容结构清晰，适合 AI 摘取
5. 每篇内容输出引用依据
6. 内容生成必须读取已确认信源发现结果
7. 内容不直接发布，需用户确认
